# Markdown 编辑器

一个轻量的 Mulby Markdown 编辑插件，默认进入接近 Typora 的普通模式，直接在可视化排版结果上编辑，并保留源代码模式入口。

## 功能

- 默认普通模式：直接在排版结果上编辑正文
- 提供源代码模式入口：切回纯 Markdown 编辑
- 基础工具栏：撤销/重做、标题、粗体、斜体、链接、引用、代码、列表、任务列表、分割线
- 撤销 / 重做：工具栏按钮 + 快捷键（`Ctrl/Cmd+Z` 撤销，`Ctrl/Cmd+Shift+Z` / `Ctrl/Cmd+Y` 重做）
- 文件工作流：新建空白文档、打开文件（绑定路径）、保存到原文件、另存为
  - `Ctrl/Cmd+S`：已打开文件时保存到该文件，未绑定文件时保存草稿；`Ctrl/Cmd+Shift+S` 另存为
  - 标题栏显示文件名、`•` 脏标记，以及当前是「文件」还是「草稿」
- 查找 / 替换：`Ctrl/Cmd+F` 查找、`Ctrl/Cmd+H` 替换，支持区分大小写、全词匹配、命中计数与上一处/下一处跳转、替换当前/全部替换
- AI 助手：**选中文字即弹出悬浮 AI 工具栏**（润色 / 翻译 / 问一问 / 续写 / 总结 / 自定义 / 生图），无选区时按 `Ctrl/Cmd+J` 也能在光标处唤起浮窗处理全文；另可 `Ctrl/Cmd+K` 打开完整面板
- AI 生图：悬浮工具栏「生图」或工具栏生图按钮打开对话框，按**划词文字或自定义提示词**调用宿主生图模型生成图片，结果一键落盘并插入文档
- 图片：工具栏插入图片、粘贴剪贴板图片、拖入图片文件；图片始终落盘以保持源码整洁（不内联 base64）：已绑定文件时存到文档同目录 `assets/` 用相对路径，未绑定文件时存到插件数据目录 `<userData>/markdown-editor/assets/` 用 `file://` 绝对路径
- 插入感知编辑器模式：图片用编辑器的 `addImage` 命令插入——**所见即所得模式下渲染为图片节点、源码模式下写入 `![](...)`**；AI 文本 / 粘贴等 Markdown 内容在所见即所得模式下也会正确渲染（含标题/列表/强调/链接等），不会把 Markdown 源码当作纯文本插入
- base64 自动整理：
  - 粘贴含内联 `![](data:image/...;base64,...)` 的 Markdown 文本时，自动把 base64 图片提取落盘并替换为短路径引用，源码不再被超长 base64 撑满
  - 工具栏「整理图片」按钮：一键把当前文档里所有内联 base64 图片转成文件引用（落盘策略同上），转换进编辑器历史可 `Ctrl/Cmd+Z` 撤销；解析或落盘失败的引用保持原样，不丢内容
- AI 助手（走宿主 `mulby.ai`）：两种入口
  - **悬浮工具栏（划词即用，推荐）**：在编辑器里选中文字后，会在选区上方自动弹出一个悬浮工具栏，直接点「润色 / 翻译 / 问一问 / 续写 / 总结 / 自定义 / 生图」即可，无需进面板
    - **`Ctrl/Cmd+J` 快捷唤起**：即使没有选中文字也能随时呼出浮窗——有选区时锚定在选区上、对选区生效；无选区时锚定在光标处、对全文生效（菜单与标题会显示「全文」标记）
    - 「问一问」仅在有选区时出现，作用是让 AI 解释选中的文字是什么、是什么意思（输出解释而非改写，结果以渲染后的 Markdown 阅读视图展示，可复制 / 插入）
    - 翻译会就地展开目标语言下拉、自定义会就地展开提示词输入框（`Ctrl/Cmd+Enter` 快速生成），全部在当前页面完成
    - 流式生成、可随时停止；结果就地预览，有选区时可「替换选区 / 在选区后插入 / 复制 / 重新生成」，无选区（全文）时提供「在光标处插入 / 复制 / 重新生成」，应用进编辑器历史可 `Ctrl/Cmd+Z` 撤销
    - 浮窗会随选区/光标自动定位（空间不足时自动翻到下方并贴边），点击别处或按 `Esc` 关闭；右侧「⤢」可一键展开为完整面板
  - **完整面板**：工具栏 ✦ 按钮或 `Ctrl/Cmd+K` 打开
    - 五种操作：润色、续写、翻译（含目标语言下拉）、总结、自定义指令
    - 润色 / 翻译需要先选中文字；续写 / 总结 / 自定义在无选区时自动处理全文
    - 可在面板里选择 AI 模型（默认用宿主默认模型），选择即持久化记忆；悬浮工具栏复用同一模型设置
  - **AI 生图**：悬浮工具栏「生图」或工具栏生图按钮打开生图对话框（走宿主 `mulby.ai.images`）
    - 两个入口：划词后点「生图」会把选中文字带入提示词；无选区时打开空白提示词由你输入；提示词均可编辑
    - 流式优先（可看到渐进预览、随时停止）；可选生图模型（自动过滤 `image-generation` 模型并记忆所选）
    - 输出尺寸以**比例**呈现，便于理解：`9:16 / 2:3 / 3:4 / 1:1 / 4:3 / 3:2 / 16:9`
    - **每篇文档的生图历史会保留**：生成后即落盘到 `<userData>/markdown-editor/gen-history/` 并按文档记录，关闭再打开对话框仍能看到本文档生成过的图片（即使从未插入）；点历史缩略图可预览、再点「插入文档」或双击缩略图即可插入
    - 点「插入文档」即把图片**落盘**（绑定文件存文档同目录 `assets/` 相对路径，未绑定时存 `<userData>/markdown-editor/assets/` 用 `file://`）并在原光标处插入引用，不覆盖用作提示词的选中文字
    - 未检测到生图模型时给出提示，引导到 Mulby「AI 设置」添加 `image-generation` 模型
  - 未配置 Mulby AI 时给出友好提示，不会崩溃
- 自动草稿保存到插件存储（未绑定文件时），也支持手动保存
- 支持从剪贴板粘贴、从本地打开 Markdown/TXT 文件
- 导出：`.md` / `.html`（内联完整样式）/ `.pdf`（无头渲染）/ `.docx`（Word）
  - Word 导出为**真实表格**（带表头底纹）与**内嵌图片**：相对 `assets/`、`file://` 与内联 base64 图片都会被解码并嵌入文档（统一转 PNG，按页宽等比缩放）；无法解码的图片降级为 `[图片: …]` 文本占位，导出永不因单张图失败
  - HTML / PDF 导出支持**代码高亮**（highlight.js）、**数学公式**（KaTeX，`$…$` / `$$…$$` / `\(…\)` / `\[…\]`）与 **mermaid 图**（```mermaid 代码块渲染为图）：通过按需注入 CDN 资源实现，不增加插件体积；仅在文档含对应内容时注入，需要联网渲染（如需完全离线可改为内置打包）
- 支持通过划词把外部文本直接带入编辑器

## 触发方式

- 关键词：`markdown` / `markdown 编辑` / `Markdown 编辑器`
- 划词动作：`用 Markdown 编辑`

## 开发

```bash
cd plugins/markdown-editor
pnpm run build
pnpm run pack
```
