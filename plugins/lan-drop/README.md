# 闪传 LanDrop

> 局域网内跨平台、零配置、拖拽即传的文件闪传工具。装进 Mulby 的「隔空投送」。

同一局域网内的设备自动相互发现，拖拽文件即可秒传，大文件全程流式直传不占内存，支持 Windows / macOS / Linux。

## ✨ 功能特性

- **零配置自动发现**：基于 UDP 广播，同一网段设备自动出现在列表中，无需输入 IP。
- **拖拽即传**：把文件/文件夹拖进窗口，或直接拖到 Mulby 搜索框选择「用闪传发送」。
- **大文件流式直传**：HTTP 之上点对点直连，边读边发边写，不把文件载入内存。
- **后台常驻接收**：开启「跟随启动」后即使不打开窗口也能接收文件（系统弹窗确认）。
- **可验证身份**：基于 x25519 身份密钥，`deviceId` 即公钥指纹（不可伪造）；仅「身份已验证 + 已信任」的设备自动接收，文件名只取 basename 防目录穿越。
- **端到端加密**：对身份可验证的对端启用 AES-256-GCM 分帧加密，会话密钥由 ECDH 协商、永不上线（可在设置中关闭）。
- **断点续传**：传输中断后重发会自动从对端已收字节处续传（基于稳定分片标识），大文件无需从头再来。
- **完整性校验**：可选 SHA-256 校验（大文件自动跳过），并按字节数核对大小。
- **手动直连兜底**：当路由器开启 AP 隔离或跨网段时，可手动输入对端 IP 直连。
- **传输队列与历史**：实时进度、速度、剩余时间，历史记录持久化。

## 🚀 使用方式

### 触发插件
在 Mulby 搜索框输入以下任一关键字打开主界面：

- `闪传`、`landrop`、`lan-drop`、`传文件`

或直接把文件拖到 Mulby 搜索框，选择「用闪传发送」。

### 发送文件
1. 打开闪传窗口（默认独立窗口）。
2. 把文件/文件夹拖进窗口，或点击拖拽区选择文件。
3. 在左侧设备列表选择目标设备。
4. 点击「发送」。对端确认后开始传输。

### 接收文件
- 保持插件运行（建议在搜索结果右键勾选「跟随 Mulby 启动」实现后台常驻）。
- 有设备发来文件时：
  - 若对方是**信任设备**或接收策略为「自动接收」→ 直接保存。
  - 否则弹出系统确认框，可选择「接收 / 拒绝 / 信任并接收」。
- 文件默认保存到 `下载目录/LanDrop`，可在设置中修改。

## ⚙️ 设置项

| 设置 | 说明 |
| --- | --- |
| 设备名称 | 本机在局域网中显示的名称（默认主机名）。 |
| 文件保存目录 | 接收文件的落盘目录（默认 `Downloads/LanDrop`）。 |
| 接收策略 | `询问确认`（默认）/ `自动接收`。信任设备始终自动接收。 |
| 启用设备发现 | 关闭后不再广播与搜索设备。 |
| 完整性校验 | 为传输计算 SHA-256（>1GiB 自动跳过）。 |
| 端到端加密 | 对身份可验证的对端用 AES-256-GCM 加密传输（默认开启）。 |

## 🔌 网络与协议

| 用途 | 协议/端口 |
| --- | --- |
| 设备发现 | UDP 广播 `52800` |
| 文件收发 / 握手 | TCP HTTP `52801` |

- 握手：`GET /mulby-landrop/info` 返回设备信息（含 `publicKey`）。
- 续传预检：`GET /mulby-landrop/offset?key=<分片标识>` 返回对端该文件已落盘字节数；发送方据此从断点续传。
- 传输：`POST /mulby-landrop/transfer`，元数据走自定义请求头（`x-ld-*`），Body 为文件流。
- 身份签名（协议 v2）：发送方附带 `x-ld-pubkey`（自证公钥）、`x-ld-ts`、`x-ld-nonce`、`x-ld-sig`（对 `transferId|deviceId|文件名|大小|ts|nonce` 的 HMAC，密钥由 ECDH 共享密钥派生）。接收方校验通过才允许「受信任自动接收」。
- 端到端加密（协议 v2，可选）：发送方附带 `x-ld-enc: aes-256-gcm` + `x-ld-enc-salt`，Body 为分帧密文 `[4字节密文长度][密文][16字节 GCM tag]`（chunked 传输）；会话密钥 = ECDH 共享密钥 + salt 经 HKDF 派生，计数器 IV、AAD 绑定 `transferId`。元数据头（文件名/大小）与签名仍为明文（用于鉴权与展示）。
- 断点续传（协议 v2）：发送方先 `GET /offset` 取对端已收字节，再用 `x-ld-resume-key`（`sha256(发送方id+文件名+大小)`）+ `x-ld-offset` 从断点发送；`content-length` 为剩余字节。接收端按偏移追加写入 `.part`，开启完整性校验时会预哈希已有分片以保证全文哈希连续。
- 首次运行时 Windows 防火墙可能弹出放行提示，请允许「专用网络」访问。

## 🔒 安全说明

- 仅服务局域网 / 本机来源（私有网段 + 回环 + 链路本地 + CGNAT），拒绝公网来源请求；默认手动确认接收，未确认前不落盘。
- 文件名只保留 basename，防止目录穿越覆盖。
- 抗滥用与容量保护：未验证请求的确认弹窗有并发上限（防刷屏 DoS）；同时接收数有上限；落盘前预检磁盘可用空间；传输一旦超出声明大小立即中止（防打满磁盘）。
- **身份自证**：`deviceId = x25519 公钥指纹`，传输签名由 ECDH 派生的共享密钥（双方各自计算、密钥永不上线）做 HMAC，配合时间戳 + nonce 防重放。「受信任自动接收」必须签名校验通过 —— 攻击者即便嗅探到广播中的 `deviceId` 也无法伪造身份绕过确认。
- 端到端加密：对身份可验证的对端，文件内容用 AES-256-GCM 分帧加密（会话密钥由 ECDH 协商、永不上线，逐帧 tag 防篡改）。被动嗅探既无法伪造身份、也无法读取内容。文件名/大小等元数据头仍为明文（用于鉴权与界面展示）。
- 回退：当对端为旧版（无公钥）时无法加密，回退为明文 + 弹窗确认；可在设置中关闭加密。
- 不支持跨网段 / VPN（设计为同一局域网直连）。

## 🧪 手动验收清单（在 Mulby 内测试）

1. 安装插件后无 manifest 报错，可在搜索框通过 `闪传` 关键字打开窗口。
2. 在设置中确认「接收服务在线 · 端口 52801」「设备发现在线」。
3. 两台同局域网设备互相能在「设备」列表看到对方（约 3 秒内）。
4. A → B 发送一个小文件：B 弹出确认框，接收后文件出现在保存目录，A/B 历史均显示「已完成」。
5. 发送一个较大文件（如 ≥200MB）：进度条、速度、剩余时间实时更新，完成后大小一致；传输中断后再次发送应从断点续传（进度不归零）。
6. 把 B 标记为信任设备后再次发送：B 不再弹窗，自动接收。
7. 关闭闪传窗口但保持插件后台运行时，仍能收到文件（系统通知 + 弹窗）。
8. AP 隔离场景：用「手动添加 IP」输入对端地址，能成功添加并发送。

## 🛠 开发

```bash
# 安装依赖
pnpm install        # 若 Windows 符号链接报错，可在本插件目录改用 npm install

# 构建（后端 esbuild + 前端 vite）
pnpm run build

# 打包为 .inplugin
pnpm run pack
```

## ⚠️ 已知限制 / 后续计划

- 文件夹发送目前会扁平化为文件列表（不保留层级，zip 打包为后续计划）。
- 已支持应用层端到端加密（AES-256-GCM）与断点续传；标准 TLS 仍可作为后续可选项。
- 断点续传按 `(发送方id, 文件名, 大小)` 标识分片：同名同大小但内容不同的文件，建议保持「完整性校验」开启（默认），以便最终 SHA-256 校验拦截异常拼接。
- 与旧版本（协议 v1，无身份密钥）互发时无法验证身份，将始终弹窗确认、不会自动接收。
- 手机扫码 Web 接收为后续计划（P2）。
