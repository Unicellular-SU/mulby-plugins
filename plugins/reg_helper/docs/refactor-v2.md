# 正则编辑器 v2.0 重构记录

## 完成时间
2026-03-18

## 重构内容

### 旧版问题
- iOS 风格侧边栏布局，在 Mulby 窗口中空间利用率低
- 功能单一，仅支持匹配，不支持替换
- 无正则速查表
- 后端存在大量冗余逻辑（与 UI 重复）

### 新版设计

#### 布局
- IDE/代码编辑器风格，紧凑布局
- 顶部工具栏 → 正则输入栏 → 可选替换栏 → 主体（左：测试文本+结果 / 右：侧面板）→ 底部状态栏

#### 新功能
1. **替换预览** - 点击工具栏"替换"按钮展开替换栏，实时预览替换结果
2. **正则速查表** - 侧面板切换到"速查表"标签，按字符类/量词/锚点/分组分类，点击直接插入
3. **交替匹配色** - 4 种颜色循环标记不同匹配项，视觉区分更清晰
4. **命名捕获组** - 支持显示 `(?<name>)` 命名捕获组信息
5. **over 触发器** - manifest 新增 over 命令，选中文本直接进入编辑器
6. **内联 Toast 通知** - 操作反馈不再依赖系统通知
7. **零宽匹配安全保护** - 防止空匹配导致的无限循环

#### 代码架构
- `src/main.ts` - 仅保留生命周期和 run 入口（~40 行）
- `src/ui/App.tsx` - 全部核心逻辑（匹配、替换、高亮、预设、速查表）
- `src/ui/App.css` - 完整样式系统（CSS 变量、暗色/亮色主题、响应式）
- `src/ui/hooks/useMulby.ts` - 复用现有 hook（未修改）
