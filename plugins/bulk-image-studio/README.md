# 图片批量工坊（bulk-image-studio）

Mulby 插件：在独立窗口中完成**批量图片处理**、**多图合并**（PDF / 横向或纵向长图 / GIF）以及**手动连续裁剪**（应用后自动切换下一张）。

## 依赖与安装说明

- 图片处理通过 Mulby 主应用提供的 **sharp API** 执行，插件包本身不携带 `sharp` 原生二进制。
- 后端仍使用 **pdf-lib**、**svgo**、**gif-encoder-2**、**to-ico** 等纯插件依赖，均已在 `package.json` 的 `dependencies` 中声明。
- 部分格式（TIFF、AVIF、ICO、GIF 等）是否可用取决于当前平台 Mulby 主应用内置的 **libvips / sharp** 构建；若单张失败，批量结果里会记录该文件错误信息。

## 功能概览

### 批量处理

- **压缩**：光栅格式质量；**SVG** 使用 **SVGO** 做结构优化（与像素压缩不同，界面中已说明）。
- **格式转换**：PNG、JPG/JPEG、WebP、TIFF、AVIF、BMP、GIF、ICO、SVG（输出 SVG 仅支持「SVG 优化」路径；其他格式由 sharp 输出）。
- **尺寸**：固定宽高（可选）、百分比缩放，`inside` / `cover` / `fill`。
- **水印**：文字（字号、颜色、不透明度、旋转、角点/居中、平铺）；图片（缩放、不透明度、旋转、位置、平铺）。
- **圆角**：按短边百分比计算半径；内部使用圆角遮罩合成。
- **补边**：上/右/下/左像素、颜色与不透明度。
- **裁剪**：按宽高比居中裁剪，可选 `center` / `north` / `south` / `east` / `west` 重力。
- **旋转**：任意角度 + 背景色。
- **翻转**：水平（左右）、垂直（上下），可组合。

**保存流程（两阶段）**：先点「开始处理」将结果写入系统**临时目录**并暂存；完成后在界面选择 **覆盖原图**、**原目录带后缀**（`原名{后缀}.扩展名`）或 **其他目录…**；也可 **放弃暂存**。覆盖时若输出扩展名与源文件不同，会写入新扩展名并删除原文件。

### 合并

- **PDF**：多页，每页一张图（非常规格式会先转 JPEG 再嵌入）。
- **长图**：横向或纵向拼接，可设间距与背景色；输出扩展名选 PNG/JPEG/WebP 等。
- **GIF**：统一画布、帧间隔、是否循环、最长边上限。

### 手动裁剪

- 队列多图，`react-easy-crop` 画布：**比例预设**（含自由随视口、自定义宽高比）、**滚轮缩放**、**三分格**开关、**重置视图**；**应用并下一张**写入 `原文件名_crop.扩展名`。
- **相对选区复用**：勾选「每次导出后用本次相对选区打开下一张」或 **从当前视图捕获模板**，便于同构图批量裁同一区域；侧栏显示已导出数量与 **跳到未导出**。
- 快捷键：**Enter** 应用并下一张、**Esc** 跳过、**Alt+←/→** 切换图片、**Ctrl±** 缩放、**Ctrl+0** 复位平移与缩放；在裁剪选框上可用 **方向键** 微调（**Shift** 更细）。

## Mulby 内触发方式

| 说明 | 触发示例 |
|------|----------|
| 批量 | 关键词：`图片批量`、`批量处理图片`、`图片批量处理` |
| 合并 | 关键词：`图片合并`、`合并 PDF`、`合并长图` |
| 手动裁剪 | 关键词：`手动裁剪`、`连续裁剪` |
| 带附件 | `files` / `img` 类型命令会附带路径，打开后进入批量页并预填队列 |

窗口为 **detached**，默认较大尺寸；`pluginSetting.single: true` 建议单实例使用。

## 调试日志

后端统一前缀 **`[bulk-image-studio]`**。在 Mulby 主进程控制台或插件宿主日志中搜索该字符串可过滤：

- **`[host]`**：`run`、`getPendingInit`、`batchProcess:incoming`、`batchCommit:incoming`、`mergePdf:incoming` 等，含 `fileSummary` / `stepsSummary`（逐项 `path` 与 `ok` 标记）。
- **`[fs]`**：每次 `readFile` / `writeFile` 前的路径；若路径非法会额外输出 **`[fs] INVALID_PATH`**（`pathType` 为 `undefined` 等）。

便于定位「`filename` Received undefined」类错误来自哪一次读写、哪张路径。

## 开发

```bash
cd bulk-image-studio
pnpm install
pnpm run build    # esbuild 后端 + Vite UI
pnpm run pack     # 生成 bulk-image-studio-1.0.0.inplugin
```

- 后端入口：`src/main.ts` → `dist/main.js`（图片处理走宿主 `mulby.sharp.execute`，插件不打包 `sharp`）。
- UI：`src/ui/` → 构建到 `ui/`。
- 可编辑图标源：`assets/icon.svg`；发布前可将导出的 **512×512** PNG 覆盖根目录 `icon.png`。

## Mulby 内验收建议

1. 分别用三条关键词打开插件，确认进入 **批量 / 合并 / 裁剪** 对应页签。
2. 选 3～5 张混合格式图片做批量压缩并转 WebP，检查输出目录与失败列表。
3. 各做一次：合并 PDF、纵向长图、GIF（帧间隔与循环）。
4. 手动裁剪连续处理 3 张，确认保存后自动切换下一张。
5. 断网环境下仍能使用（无额外联网依赖）。
