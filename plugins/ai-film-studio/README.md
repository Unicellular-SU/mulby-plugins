# AI 影视工坊（ai-film-studio）

> 无限画布 · 故事到影片 · 多模态 AI 工作流

在 Mulby 的无限画布上，用**可视化节点工作流**把"一段话 / 一个故事"逐步加工成短片：

```
故事 → 剧本 → 分镜 → 角色图/场景图 → 关键帧 → 视频片段 →（配音/字幕/合成）→ 成片
```

文本与图像默认复用 Mulby 宿主 AI（零配置）；视频由插件自管第三方供应商（如 fal.ai）。每个能力都是画布上的一个节点，连线即数据流，可分支、可重跑、可锁定。

---

## ✨ 功能特性

- **无限画布工作流**：基于 React Flow，拖拽添加节点、连线编排、缩放平移；连线带端口类型校验（`text/json/image/video/audio`）。
- **文本链路（零配置）**：故事梳理 → 剧本 → 分镜脚本，调用 `mulby.ai.call` 流式生成，结构化 JSON 输出。
- **图像链路（零配置）**：角色三视图 / 场景概念图 / 关键帧，调用 `mulby.ai.images`；支持参考图（img2img，`ai.images.edit`）保持角色一致性。
- **视频链路（自管供应商）**：图生视频 / 文生视频，统一异步三段式 `submit → poll → fetch`；内置 `fal` 与 `custom-http` 适配器；生成后自动下载落盘到本机。
- **多供应商管理**：增删供应商、节点级模型/供应商覆盖、连通性自测；API Key 走系统级加密存储（Keychain/DPAPI），结构中无明文。
- **工程持久化**：防抖自动保存、多工程切换、JSON 导入导出；图像存附件存储、视频落 `userData` 文件系统。
- **短剧多集工作台**：同一项目可维护多集；每集独立保存剧本、分镜、视频候选和时间线，角色/场景/道具资产在项目级共享复用。
- **资产变体与一致性守门**：同一角色可建立时期、妆容、服装、受伤状态等变体；资产支持别名/昵称/称号，Agent 和分镜出场匹配会优先复用同一资产；分镜用 `castRefs` 锁定具体变体，生成前检查缺失参考图、跨集适用范围、本集专属形态未绑定、章节分配、未引用资产、同类型资产重名/别名冲突和连续场景复用问题。
- **跨集承接建议**：Episode 切换器可查看最近制作回顾和当前集复用过的资产/形态，并给出补参考图、标记适用本集、新建并应用本集形态等建议；可一键处理可执行项，新形态会基于上一相关剧集状态预填提示词；Agent 也可通过 `get_episode_handoff` 读取同一套线索。
- **分集/全季导出**：单集可导出成片、旁挂 SRT 和 `episode.json`；多集项目可把已成片剧集批量复制到一个导出目录，并生成每集旁挂 SRT 与 `manifest.json`。导出清单包含集号、标题、源路径、导出路径、字幕文件、素材引用表和缺失项报告。
- **主题跟随宿主**：暗色优先，自动适配 Mulby 明/暗主题。

## 🧩 节点一览

| 分类 | 节点 | 输入 → 输出 | 说明 |
|---|---|---|---|
| 输入 | 故事输入 / 文本片段 | — → text | 原始素材 |
| 输入 | 参考图 / 音频素材 | — → image/audio | 本地上传，作为下游参考/音轨 |
| 输入 | **人物 / 场景** | — → json + image | 角色/场景**资产**：「运行此节点」文字生成（三视图/概念图）**或**「上传图片」用本地图；身份(JSON)+参考图直连关键帧，跨镜一致 |
| 文本 | 剧本生成 | json/text → json | 分场剧本 |
| 文本 | 分镜脚本 | json → json | 镜头表（景别/运镜/时长/角色/场景） |
| 文本 | 角色设定 | json/text → json | 角色描述 + 三视图提示词 |
| 文本 | 提示词处理 | text → text | 扩写 / 中英互译 / 风格化 |
| 图像 | 角色三视图 | json → image(扇出 N) | 多角色批量三视图 |
| 图像 | 场景概念图 | json/text → image(扇出) | 场景设定图 |
| 图像 | 分镜关键帧 | shot + chars + 参考图 → image(扇出) | 单镜首帧（参考图按角色名匹配 + 场景全收） |
| 图像 | **图生图/重绘** | image + image(参考) + text → image | 重绘/风格迁移/局部修改；多参考图条件生成 |
| 图像 | **高清重绘** | image → image | 按原内容重绘并增强细节（经图像模型，非纯像素放大） |
| 视频 | 图生视频 I2V | image(首帧) + image(尾帧,可选) + text → video | 关键帧动起来；连尾帧即首尾帧约束补间 |
| 视频 | 文生视频 T2V | text → video | 纯文本生成片段 |
| 音频 | 配音 TTS | text → audio | OpenAI 兼容语音合成（Key 加密存储） |
| 音频 | **配乐 BGM** | text → audio | 异步供应商框架生成背景音乐（需 custom-http 音乐端点） |
| 输出 | 预览 | any | 内联预览文本/图/视频/音频 |
| 输出 | **合并/收集** | any×N → any(合集) | 把多路同类产物收成一组喂下游 |
| 输出 | 影片合成 | video×N + audio + json(分镜) → video | ffmpeg 归一+拼接，可选烧录/软字幕 + 配音混音 |
| 输出 | 导出 | video | 另存成片到本机指定位置 |

> 全局画风/画幅由顶栏 🎨「全局设定」统一注入所有生成节点（无需画布连线）。合成节点的「视频片段」端口可连多个视频节点（或「合并」节点），按拓扑顺序拼接；字幕由分镜 JSON 按片段时长生成。首次合成会自动下载 ffmpeg。

## 🚀 使用方法

1. 在 Mulby 中通过关键词 `故事画布` / `AI视频` / `分镜` / `影视工坊` 打开插件（独立窗口）。
2. 从左侧节点库拖拽节点到画布，按数据流连线（如 `故事输入 → 剧本生成 → 分镜脚本`）；或点顶栏「＋模板…」一键载入预置流水线（故事→分镜 / 完整影视流水线 / 片段→成片）。
3. 顶栏选择文本/图像模型（复用你在 Mulby 已配置的模型）。
4. 点击节点的「运行此节点」或顶栏「运行」执行全图（按拓扑顺序）。
5. 在右侧属性面板查看结果（JSON / 大图 / 视频播放器），可逐节点重跑。

### 短剧多集工作流

1. 在工作台导入原著或输入故事后，先用「原著」页拆章、提取章节事件，并按目标集数创建剧集。
2. 用 Episode 切换器进入某一集，生成或编辑本集剧本、分镜、关键帧、视频候选和时间线；切换剧集时，当前视图只展示本集内容。
3. 资产页维护全剧共享角色、场景、道具；同一角色、场景或道具可补充别名/昵称/称号，避免后续剧集因为称呼变化重复建资产；同一角色的不同妆容、服装、年龄阶段用「形态/妆容」变体管理，并标记适用剧集。
4. 分镜里选择出场资产时可绑定具体变体；关键帧和视频生成前会校验资产/变体参考图，不让缺失的妆容或时期图静默进入生成链路。
5. 分镜页的一致性检查会汇总多集章节覆盖、重复分配、变体跨集适用范围、本集专属形态未绑定、项目级资产未被分镜引用、同类型资产重名/别名冲突、连续场景资产复用和缺失参考图，并提供常见问题的一键修复；唯一候选形态可直接一键绑定到当前集分镜。同一 `sceneId` 的连续分镜会检查是否复用同一个场景资产；Agent 解析出场名单时会按资产名称和别名匹配，尽量复用全剧资产池中的既有身份。
6. Episode 切换器旁的承接入口会汇总最近制作回顾，以及当前集角色/场景/道具形态在其他集里的出现记录；处理续集、换装或妆容变化前先看这里，并可一键补图、标记形态适用本集或新建本集形态。新建本集形态时会基于上一相关剧集状态预填形态提示词。
7. 顶栏「生成全剧」会按剧集顺序逐集执行资产、被分镜引用的形态参考图、关键帧、视频和合成；已成片、失败和暂缓剧集默认跳过，运行中可请求“暂停后续”并在当前集完成后停止，每集成片路径和制作回顾会回写到对应 Episode，供后续集生成时引用。
8. Episode 切换器里的导出按钮会把当前集成片、旁挂 SRT 和 `episode.json` 打成单集包；顶栏「导出全季」会把已成片剧集复制到你选择的目录，并为每集写入旁挂 SRT 和 `manifest.json`。两种清单都会附带字幕文件、素材引用表和缺失项报告，方便交付、复查或继续剪辑。
9. 需要重跑某集时，可在 Episode 切换器里重置当前集成片状态；尚未准备好的剧集可先标记为暂缓，之后恢复进入「生成全剧」队列，不必靠随手改分镜触发失效。

### 配置模型供应商（视频 / 配乐 / 语音）

Mulby 不内置视频/配乐/语音模型，需自管供应商。**一个供应商可声明多种能力**，节点按能力选用、可在节点上覆盖：

1. 顶栏点击「模型供应商」打开设置弹窗。
2. 选一个**预设**快速填充：国内直连——**火山方舟 Seedance/豆包**、**阿里百炼 通义万相**；**OpenAI 兼容视频聚合**——**toapis（Sora2 / Veo3 / Grok，统一 `/v1/videos/generations`）** + 通用兼容预设；其它**聚合平台**（302.AI / 云雾 / 硅基流动）；**OpenAI 兼容语音**；海外 fal 视频/配乐。或手动配置：
   - 勾选**能力**：视频 / 配乐 / 语音（模式自动：语音=同步返回音频；视频/音乐=异步轮询）。
   - **fal**：填 `model`（如 `fal-ai/...`）+ API Key，一个 Key 覆盖 Kling/Veo/Sora/Seedance 等。
   - **custom-http**：填 `submit` / `poll` URL 与结果字段 JSON 路径（留空按常见命名自动尝试），兜底任意异步供应商。
   - **语音(同步)**：填接口地址（如 `https://api.openai.com/v1`）+ 模型(如 `tts-1`)+ 音色列表。
3. 填 API Key（密文存储），异步供应商可点「测试连接」自测；点单选圈把它设为对应能力的**默认**。
4. 回画布：关键帧→「图生视频」、文本→「文生视频/配音TTS」、描述→「配乐BGM」，运行即用对应能力的默认供应商（或节点覆盖）。

> **API Key 安全**：所有 Key 通过 `storage.encrypted`（macOS Keychain / Windows DPAPI / Linux Secret Service）加密存储，工程与供应商结构里只保存键名引用，绝不落明文。

## 🛠 开发与构建

```bash
npm install        # 安装依赖
npm run build      # 构建：esbuild 后端(dist/main.js) + vite 前端(ui/)
npm run pack       # 打包为 .inplugin（需要 mulby CLI）
```

- 后端 `src/main.ts`：视频下载落盘、配音合成（M5）、文件导出等 RPC。
- 前端 `src/ui/`：画布、节点、引擎与供应商适配器。AI 文本/图像调用放前端（避免 IPC 超时、便于流式）。

### 目录结构

```
src/
  main.ts                  # 后端：下载/落盘/导出 RPC
  types/mulby.d.ts         # 宿主 API 类型
  ui/
    App.tsx  main.tsx  styles.css  index.html
    nodes/nodeDefs.ts      # 节点定义（端口/参数/图标）
    store/                 # graphStore（图+运行+持久化）/ providerStore
    components/            # FlowCanvas / Toolbar / NodeLibrary / Inspector / ProviderSettings / nodes
    services/
      textEngine / imageEngine / models / prompts / jsonParse / executor
      assets（storage.attachment）/ keys（storage.encrypted）/ download
      providers/           # fal / customHttp / http / test / index(runVideo)
```

## ✅ Mulby 内手动验收清单

- [ ] 关键词能打开插件，画布可拖拽/连线/缩放/平移。
- [ ] `故事输入 → 剧本生成 → 分镜脚本` 连线后运行，得到结构化剧本与分镜表。
- [ ] 选择图像模型，由分镜/角色设定生成关键帧 / 三视图 / 场景图，节点显示缩略图。
- [ ] 上传参考图接入关键帧，触发 img2img（一致性）。
- [ ] 配置 fal 供应商并「测试连接」通过；关键帧 → 图生视频，节点内联播放。
- [ ] 创建多集项目，原著章节能按集分配；切换剧集后剧本、分镜、视频候选互不覆盖。
- [ ] 同一角色能新增妆容/时期变体，分镜可选择变体；缺失变体参考图时生成前置检查会阻止继续。
- [ ] 分镜页一致性抽屉能显示章节覆盖、重复章节、未引用项目资产、同类型资产重名/别名冲突、连续场景资产复用、变体适用范围、本集专属形态未绑定和缺失参考图，并能执行对应快速修复；资产别名能被搜索、Agent 资产写入和分镜出场匹配复用。
- [ ] Episode 切换器旁的承接入口能显示最近制作回顾和当前集复用资产/形态，并能一键执行补参考图、标记适用本集、新建并应用本集形态；新建形态会带上一相关剧集状态提示词；Agent 工具能读取 `get_episode_handoff`。
- [ ] 顶栏「生成全剧」能逐集生成资产、分镜实际引用的形态参考图并合成，已成片、失败和暂缓剧集会跳过；运行中可暂停后续剧集；多集输出文件名包含集号和集标题，切回对应剧集能看到「已成片」「成片失败」或「已暂缓」状态。
- [ ] 当前集已成片后，Episode 切换器里的导出按钮能导出成片、旁挂 SRT 和包含剧本、分镜、字幕文件、素材引用表、缺失项报告的 `episode.json`。
- [ ] 顶栏「导出全季」能选择目标目录，复制已成片剧集和每集旁挂 SRT，并生成包含集号、标题、源路径、导出路径、字幕文件、素材引用表和缺失项报告的 `manifest.json`。
- [ ] 当前集已成片或成片失败后，Episode 切换器能重置该集成片状态；重置后该集可重新进入「生成全剧」队列。
- [ ] 视频生成后自动落盘，属性面板可「打开文件夹」。
- [ ] 配音节点填入 TTS API Key（加密保存），由文本合成音频并播放。
- [ ] 多个视频片段 + 分镜(字幕) + 配音 → 影片合成节点「运行」，首次自动下载 ffmpeg，得到带字幕/配音的成片。
- [ ] 导出节点把成片另存到指定位置。
- [ ] 新建/切换/重命名/删除工程；导入导出 JSON；重开后图像与工程恢复。
- [ ] 切换 Mulby 明/暗主题，界面跟随。

## 📌 依赖与前置

- Mulby 宿主（提供 `ai` / `storage` / `http` / `filesystem` / `security` 等 API）。
- 文本/图像：需在 Mulby 设置中已配置至少一个文本模型与图像模型（否则相应节点不可用）。
- 视频：需用户自带第三方供应商账号与 API Key。
