# AI 创意画布（ai-creative-canvas）

> 无限自由画布上的 AI 影像创作 + 媒体编辑工坊。在一张可无限平移缩放的画布上，以「卡片」为单位**逐张**生成图像 / 文本 / 视频 / 配音，并内置一套**逐卡片媒体后期工具箱**（裁剪 · 扩图 · 抠像 · 放大 · 宫格分镜 · 视频裁剪 · 转 GIF · 抽帧 · 镜头检测 · 音视频分离 · 倒放 · 压制）。

画布引擎**完全自研**（DOM 卡片 + 单一 CSS transform 世界层 + SVG 连线 + 自写图状态，参考 AI-CanvasPro 的实现路线），不依赖任何第三方画布库。

## 与 ai-film-studio 的区别

| | ai-film-studio | **ai-creative-canvas（本插件）** |
|---|---|---|
| 范式 | 故事→分镜→成片的**自动流水线**（一键 Run All） | **自由画布 + 逐卡片按需生成 + 富媒体后期编辑** |
| 画布 | React Flow 流程图 | 自研无限白板（自由摆放/缩放/连线/多画布） |
| 重心 | 编排自动化 | 创作探索 + 媒体编辑工具箱 |

两者定位互补，互不冲突（独立的 id、触发词、存储与落盘目录）。

## 触发方式

在 Mulby 搜索框输入以下任一关键词：

- `创意画布` · `无限画布` · `AI画布`

或把图片 / 视频 / 音频文件拖到 Mulby（`img` / `files` 触发），会直接在画布中作为素材卡打开。

## 功能

- **无限画布**：滚轮缩放（朝光标）、空格 / 中键拖拽平移、框选多选、拖动、网格、小地图、适配视图、撤销/重做、多画布 Tab。
- **卡片**：文本 / 图片 / 视频 / 音频 / 素材。拖拽、粘贴、拖入文件即可导入素材。
- **AI 生成**（Mulby 原生）：
  - 文本卡：`ai.call` 流式生成（分镜脚本 / 文案 / 提示词扩写），支持看图（引用图片作 vision 输入）。
  - 图片卡：`ai.images` 文生图（流式进度 + 预览）、图生图、多图参考一致性（`referenceAttachmentIds`）。
- **引用**：把一张卡片连线到另一张（拖卡片右侧圆点），或在右侧 Inspector 用下拉添加引用；生成时自动把被引文本/图片作为上下文/参考。
- **图像媒体工具箱**（`sharp` + `ai.images.edit`）：裁剪（框选）、扩图、放大、抠像/去背景、宫格分镜切分（2×2 / 3×3 / 1×3）。
- **视频媒体工具箱**（`ffmpeg`，首次自动下载）：裁剪片段、转 GIF、抽帧、镜头检测（取每个切点代表帧）、提取/去除音轨、倒放、压制。
- **可插拔视频/音频生成**（外部 Provider，右上角「设置」配置）：
  - 视频：通用「异步 submit + poll」，按 JSON 路径映射任务 id / 状态 / 结果 URL（兼容 fal、Replicate 风格），支持图生视频。
  - 音频：OpenAI 兼容 `/audio/speech` 配音。
  - API Key 经系统安全存储加密保存。
- **持久化**：工程（画布/卡片/连线/视口）自动保存到 Mulby storage；媒体文件落本地磁盘（`<userData>/ai-creative-canvas/media/<projectId>/`）。
- **导出**：选中带媒体的卡片，点 Inspector 的下载图标导出到任意路径。

## 前置条件

1. **AI 模型**：在 Mulby「AI 设置 → 模型管理」配置：
   - 至少一个文本模型（文本卡）。
   - 至少一个 **image-generation** 端点类型的图像模型（图片卡 / 扩图 / 放大 / 抠像）。图片卡的模型选择器会自动选中第一个可用图像模型。
2. **FFmpeg**：视频工具首次使用时会自动下载 FFmpeg（带进度提示），无需手动安装。
3. **视频/音频生成 Provider（可选）**：右上角齿轮「设置」新建并配置 Provider + API Key。不配置也不影响核心功能（文/图生图、全部媒体编辑、文本均为 Mulby 原生、零外部密钥）。

## 技术栈

- 前端：React 18 + 自研画布引擎 + zustand（状态）+ Tailwind（面板样式），Vite 构建。
- 后端（host-worker，`src/main.ts`）：仅承担渲染进程不便/易截断的工作 —— 远程媒体落盘（规避 CORS）、图床 multipart 上传、OpenAI 兼容 TTS 二进制合成、导出落盘。
- Mulby Host API：`ai`（文本/图像/附件）、`sharp`、`ffmpeg`、`http`、`storage`（含 encrypted）、`filesystem`、`system`、`notification`、`theme`、`host`。

## 开发

```bash
# 安装依赖（pnpm workspace）
pnpm install

# 构建（esbuild 后端 + vite UI）
npm run build

# 打包为 .inplugin
npm run pack
```

## 已知限制

- Mulby 内置 AI 无视频生成、TTS 仅即时播放（不产文件），故**视频生成与配音文件依赖外部 Provider**（已做成可插拔）。
- 镜头检测输出的是每个切点的代表帧（用于快速搭分镜），不直接产出按时间切好的片段。
- `@` 引用目前通过「连线 + Inspector 下拉」表达；提示词内 `@` 自动补全 pill 为后续增强项。
