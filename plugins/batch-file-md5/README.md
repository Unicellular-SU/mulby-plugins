# 批量文件 MD5（batch-file-md5）

在 Mulby 内为**单个或多个本地文件**计算 MD5。后端使用 **Node 流式读取**（约 1MB 缓冲）避免大文件占满内存，并对多文件做 **有限并行**（默认 6 路，上限 32），在常见 SSD 上能较快跑完批量任务。

## 功能与命令

| 触发方式 | 说明 |
|----------|------|
| 关键词：`文件MD5`、`批量MD5`、`MD5校验`、`计算MD5` | 打开插件面板 |
| 命令类型：文件（可多文件） | 从启动器附带文件打开并自动加入队列 |

## 使用说明

1. 点击 **选择文件**，在系统对话框中多选文件；或将文件 **拖入** 插件窗口（需 Electron 提供的本地路径，一般从资源管理器拖入有效）。
2. 点击 **开始计算**，结果逐行显示：MD5、文件大小；出错或文件夹会显示原因。
3. **复制全部**：按 `md5  完整路径` 格式复制多行（类似 `md5sum` 列表）；单行可用每行右侧的复制按钮。

## 技术说明

- **UI**：拟态（内外阴影）+ 磨砂玻璃（`backdrop-filter`），支持 Mulby 明暗主题。
- **后端**：`crypto.createHash('md5')` + `fs.createReadStream`，无额外原生依赖，便于打包为 `.inplugin`。

## 构建

```bash
cd plugins/batch-file-md5
npm install
npm run icon    # 可选：从 assets/icon.svg 生成根目录 icon.png（512×512）
npm run build
npm run pack
```

## 在 Mulby 中验收

1. 附加开发目录或安装打包后的 `.inplugin`，启用插件。
2. 用关键词打开面板，选择 1 个小文件，结果应为 32 位小写十六进制 MD5。
3. 多选多个文件，确认耗时与结果条数一致；故意加入一个文件夹路径应显示跳过或错误提示。
4. 从支持「文件」类命令的入口附带多个文件打开，确认列表自动带入。
