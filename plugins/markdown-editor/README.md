# Markdown 编辑器

一个轻量的 Mulby Markdown 编辑插件，基于 **CodeMirror 6** 打造 **Obsidian 式单窗格实时预览**：原始 Markdown 即数据源，渲染为成品、**光标所在处自动露出原始语法**供编辑，无需在「源码 / 渲染」之间手动切换。**逐 token 露出**：同一行包含多种语法时，只有鼠标点中的那一段（如某个 `**粗体**`）切回源码，本行其余部分保持渲染（对标 Obsidian）。

## 功能

- Obsidian 式实时预览：边打边渲染，光标停留的那一行显示原始 Markdown，移开即渲染。覆盖的语法与美化（均适配明暗主题）：
  - 标题、强调（粗体/斜体/删除线）、行内代码、链接、图片、分隔线、任务列表
  - **无序列表**：`-` / `*` / `+` 渲染为圆点 `•`（有序列表保留编号）
  - **列表折叠**：含子项 / 续行的列表项可折叠/展开（Obsidian 式）——鼠标移到列表项上时其左侧出现折叠箭头，点击折叠，折叠后显示 `…` 占位，再点箭头或 `…` 展开；也支持折叠快捷键（`Ctrl/Cmd+Shift+[` / `]`）
  - **引用**：隐藏 `>` 标记，整段渲染为带强调色竖条 + 浅色底纹（支持多行）；**嵌套层级**（`>`、`> >`、`> > >`）以「每级一条竖条 + 递进缩进」清晰区分（竖条采用 rem 度量，引用内代码块/表格等较小字号也不会错位）；**引用内嵌套的块级语法**（代码块、表格、`$$`公式、Mermaid、定义列表/`<details>`）会自动剥离 `>` 前缀正确渲染，并由块部件自绘同级竖条 + 缩进，使引用竖条贯穿整块不断裂（引用内表格为只读渲染，点击可展开源码编辑）
  - **代码块**：带边框圆角的代码块容器 + 真正的**语法高亮**（按语言懒加载解析器，颜色随主题切换）；渲染时隐藏上下 ` ``` ` 围栏，**语言名显示在右上角作为标签**，光标进入代码块时再露出围栏以便编辑
  - **表格**：GFM 表格渲染为圆角真实表格（表头底纹、斑马纹、横向滚动），并支持 **Obsidian 式就地编辑**——表格**始终保持渲染**，单击不再切换整块源码（避免来回切换造成的布局抖动）：直接在渲染后的单元格上编辑，单元格获得焦点时就地切换为原始文本编辑、失焦后立即渲染；`Tab` / `Shift+Tab` / `Enter` 在单元格间移动，`Esc` 取消本格编辑。鼠标移到表格上时显示控件：右侧 `＋` 在末尾加列、底部 `＋` 在末尾加行；移到列头/行首显示该列/行的删除按钮与拖动手柄（拖动可**调整行 / 列顺序**），列头还有**对齐方式切换**按钮（默认/左/中/右循环）。所有编辑都会写回 Markdown 源码（可 `Ctrl/Cmd+Z` 撤销）；格式异常无法解析的表格会保留原始源码以便手动修复
  - **数学公式**：`$…$` 行内、`$$…$$` 块级用 **KaTeX 内置渲染**（离线、无需联网）
  - **高亮**：`==文本==` 渲染为荧光笔高亮（Obsidian 风格）
  - **内联 HTML（安全子集）**：`<sup>`/`<sub>` 上下标、`<mark>` 标记、`<u>` 下划线、`<kbd>` 按键样式、`<span style="…">` 内联样式（仅白名单 CSS 属性，过滤 `url()`/`expression()`/`javascript:` 等危险值，经 setAttribute 应用而非 innerHTML）、`<!-- 注释 -->`（单行隐藏 / 多行弱化）、`<img src width height>` 按尺寸渲染、`<dl><dt><dd>` 定义列表（均以文本安全构建，不执行任意 HTML）
  - **自动链接 / 邮箱**：`<https://…>`、`<a@b.com>` 渲染为可点链接（隐藏尖括号）；**引用式链接定义** `[ref]: url "title"` 弱化为元信息
  - **脚注**：`[^1]` 渲染为上标角标，`[^1]: …` 定义行标记
  - **emoji 短码**：`:smile:` 等常用短码渲染为 emoji
  - **Mermaid**：` ```mermaid ` 代码块渲染为流程图（懒加载 mermaid，按需加载、不占主包；随明暗主题切换；语法错误时回退显示源码）
  - **折叠块**：`<details>` / `<summary>` 渲染为可点击展开/收起的折叠区（summary 以纯文本安全构建，正文按 Markdown 渲染）
  - **引用内代码块 / 复杂嵌套列表**（含代码块、引用、表格）均可组合渲染；列表内的**块级部件**（表格、`$$` 公式、Mermaid、定义列表、`<details>`、独立图片）会按其所在列表层级**缩进对齐到列表项之下**，不再跳回最左侧
  - **图片**：独立成行的图片渲染为图块；**编辑该行时不隐藏图片**，而是在图片上方露出 `![](…)` 源码、图片仍显示在下方，避免显示/隐藏图片造成的布局跳动
- 链接交互（Obsidian 式）：**单击**触发、**双击**进入源码编辑；**Cmd/Ctrl + 点击**亦可立即触发。单击动作按类型区分：
  - 外部链接 / 自动链接 / 邮箱 → 在系统浏览器打开
  - `#锚点`链接 `[文字](#标题)` → 跳转到文档内对应标题（GitHub 式 slug 匹配，支持中文标题）
  - 脚注引用 `[^1]` → 跳转到对应的 `[^1]:` 定义
- 右键菜单（**上下文感知**、跟随明暗主题、支持子菜单与键盘导航）：在编辑器内右键，按点击位置弹出对应菜单
  - 有选区：剪切 / 复制 / 粘贴 · 格式▸（加粗 / 斜体 / 删除线 / 行内代码 / 高亮）· 转换为▸（标题 1-3 / 引用 / 无序·有序·任务列表 / 代码块）· 转为链接 · 全选
  - 光标在**链接**上：打开链接 / 复制链接地址 / 编辑链接 / 取消链接（保留文字）
  - 光标在**图片**上：复制图片地址 / 打开预览 / 删除图片
  - 光标在**表格**内：在上/下方插入行、在左/右侧插入列、删除本行/本列、本列对齐（在点击位置就地插入，补足悬停控件只能在末尾追加的不足）
  - 无选区：粘贴 / 全选 · 插入▸（标题 / 列表 / 任务 / 引用 / 代码块 / 表格 / 分隔线 / 链接 / 图片 / 公式块）· **AI 工具条**（唤起划词浮窗，等同 `Ctrl/Cmd+J`）· 查找 / 替换
- 精简顶部工具栏：撤销 / 重做、打开文件、保存、另存为、导出、AI 助手（可折叠隐藏）。格式 / 转换 / 插入 / 剪贴板等均改由**右键菜单**与**直接输入 Markdown 语法**完成；整理图片、AI 生图 也在右键菜单（无选区时）
- 撤销 / 重做：工具栏按钮 + 快捷键（`Ctrl/Cmd+Z` 撤销，`Ctrl/Cmd+Shift+Z` / `Ctrl/Cmd+Y` 重做）
- 文件工作流：新建空白文档、打开文件（绑定路径）、保存到原文件、另存为
  - `Ctrl/Cmd+S`：已打开文件时保存到该文件，未绑定文件时保存草稿；`Ctrl/Cmd+Shift+S` 另存为
  - 标题栏显示文件名、`•` 脏标记，以及当前是「文件」还是「草稿」
- 多标签（VS Code / Obsidian 式，编辑器顶部标签栏，跟随明暗主题）：
  - 同时打开多个文档，每个标签**独立保留撤销/重做历史、光标与滚动位置**（基于 CodeMirror `EditorState` 快照，单视图按标签切换状态）
  - 点击文件树 / 「最近」里的文件：已打开则**激活**对应标签，否则**新开**标签；标签栏右侧 `＋` 或工具栏「新建」开未命名标签（`Ctrl/Cmd+T` 亦可）
  - 标签显示文件名与**脏标记圆点**（悬停变关闭按钮）；点关闭按钮或**中键**关闭；关闭有未保存改动的标签会弹「保存 / 不保存 / 取消」确认
  - 关闭当前标签后自动激活相邻标签；关掉最后一个标签会保留一个空白未命名标签
  - 文件树「当前文件高亮」跟随活动标签；在树里**重命名/删除**已打开文件会同步对应标签（改名更新标题，删除关闭标签）
  - **会话恢复（hot-exit）**：重开插件自动恢复上次打开的所有标签与活动标签；未保存的未命名 / 已改动标签会保留其内容（即使从未存盘也不丢失），已保存的文件标签从磁盘重新载入；任意编辑 / 开关 / 切换标签都会防抖自动持久化整套会话，窗口隐藏或关闭前再兜底写一次
  - **标签管理打磨**：**拖拽重排**标签顺序（落点显示竖条指示）；快捷键 `Ctrl/Cmd+T` 新建、`Ctrl/Cmd+W` 关闭当前、`Ctrl/Cmd+PageUp/PageDown` 或 `Ctrl/Cmd+Alt+←/→` 切换上一/下一个标签（`Ctrl/Cmd+Tab` 会被系统占用故不采用，plain `Ctrl/Cmd+方向键` 是编辑器的光标移动故加 `Alt`）；**标签右键菜单**（关闭 / 关闭其他 / 关闭全部 / 复制路径 / 在文件树中显示）；标签过多时**横向溢出滚动**（滚轮直接横滑、活动标签自动滚入可见区）。「关闭其他 / 全部」会**保留有未保存改动的标签**以防误丢，并提示保留数量
- 文件管理器（左侧「文件 | 大纲」标签切换，VS Code / Obsidian 式，跟随明暗主题）：
  - **多工作区根**：可添加多个文件夹（VS Code 多工作区式），每个根显示为可折叠的「文件夹头」、可单独「从工作区移除」（拖拽 / 右键 / 新建均按所属根）；文件树**懒加载**（展开目录时才读取子项），只显示 `.md/.markdown` 文件（文件夹照常显示以便导航）
  - **草稿（自动保存）**：随手新建的未命名文档一旦有内容，会自动落地为 `<userData>/markdown-editor/drafts/` 下的真实 `.md`（按首行命名），并在**固定置顶的「草稿」根**中列出——编辑实时写盘、可浏览/搜索、永不丢失；该根不可移除
  - 点击文件在编辑器打开并**高亮当前文件**；顶部「最近」列出最近打开的文件 / 文件夹（持久化、点击直达、可移除）
  - 工作区根集合、展开状态、最近列表、侧栏标签均持久化，重开插件自动恢复（旧版单根配置自动迁移到多根）
  - **新建文件 / 文件夹**：工具栏按钮或右键 → 树内**就地输入命名**（回车确认 / Esc 取消；新建文件名无后缀自动补 `.md`；重名自动加序号；新建文件后自动打开）
  - **重命名 / 删除**：右键就地重命名（联动当前打开文件的标题与路径）；删除统一**移到系统废纸篓**（可恢复，删前二次确认）
  - **剪切 / 复制 / 粘贴**、**创建副本**，以及**拖拽移动**（拖到文件夹或树空白处=移到根；落点虚线高亮；自动阻止把文件夹拖进自身/子目录）
  - **文件监听**：外部对已展开目录的新增 / 删除 / 改名会自动刷新（基于 preload 的 `fs.watch`，仅监听展开目录、跨平台安全）
  - 右键菜单还包含「在系统中显示」「复制路径」「刷新」
  - **键盘导航**（文件树获得焦点时）：`↑/↓` 移动、`→` 展开或进入子项、`←` 折叠或回到父级、`Enter` 打开文件 / 展开折叠、`Home/End` 跳首尾、字母键 **typeahead** 快速定位；工具栏「定位当前文件」会展开并滚动到当前文件、「折叠全部」一键收起所有目录；打开文件时自动定位到树中的它
  - **大目录虚拟滚动**：可见行数较多时只渲染视口内的行，保证滚动流畅
  - 实现说明：文件系统能力由插件根目录的 `preload.cjs`（注入 `window.mdeFs`，Node `fs/path` 桥）提供，绕过渲染层对非对话框路径的沙盒授权限制；UI 仅通过 `services/fsBridge.ts` 适配层访问，桥不可用时自动降级提示
- 查找 / 替换：`Ctrl/Cmd+F` 查找、`Ctrl/Cmd+H` 替换，支持区分大小写、全词匹配、命中计数与上一处/下一处跳转、替换当前/全部替换
- AI 助手：**划词结束后**（鼠标松开）在选区上方弹出**紧凑悬浮工具栏**——常用操作直接显示，其余收进「更多」下拉，既省空间又一键可达（润色 / 翻译 / 问一问 / 续写 / 总结 / 自定义 / 生图）；无选区时按 `Ctrl/Cmd+J` 也能在光标处唤起浮窗处理全文；另可 `Ctrl/Cmd+K` 打开完整面板
- AI 生图：悬浮工具栏「生图」或右键菜单生图打开对话框，按**划词文字或自定义提示词**调用宿主生图模型生成图片，结果一键落盘并插入文档
- 行内 AI 补全（ghost text，Copilot 式）：开启后，输入**停顿**约 0.5s 会在光标处给出**灰色续写建议**——`Tab` 接受全部、`Ctrl/Cmd+→` **逐词接受**（中文逐字）、`Esc` 忽略。建议会以**打字机效果逐字流式显示**（边生成边显示；即使模型一次性返回，也会平滑地逐字「打出来」而非整段突然弹出），完整文本始终在内部保留——所以即便还在逐字显示，`Tab`/打字顺接也作用于完整建议。等待模型时会在光标处显示**小型加载动画（三个跳动的点）**，明确提示「正在生成」，开始逐字显示即消失。**打字顺接**：你打的字若与建议一致会自动消费该建议、不重复请求；**中文输入法（IME）下同样生效**——以合成确认后的最终文字（而非中途拼音）来判断是否顺接，输入拼音期间灰字暂隐、选定汉字后立即续上，不会被误判为不一致而重新请求。**默认关闭**；中文输入法合成期不打扰；**一旦移动光标或继续输入触发新的补全，正在执行的上一个请求会被立即中止**（不浪费额度，旧的加载/灰字随即消失、在新位置重新开始）。**开关与模型在 AI 面板（`Ctrl/Cmd+K`）右上角齿轮「AI 设置」里单独成页**：含「行内 AI 补全」开关，以及独立的**「行内补全模型」**（与主模型解耦）。逐字补全延迟敏感，**建议选快速的普通模型**；推理模型（由宿主依据 **models.dev** 能力数据判定，下拉标注「推理 · 较慢」）会先思考再出字、较慢，选中时会提示；**若仍坚持用推理模型，补全会自动请求关闭其「思考」**（对支持开关的模型如 deepseek-v4 生效，宿主会把 `thinking:{type:disabled}` 透传给 provider）。留空＝跟随主模型。（编辑器内右键也可快速开关）
- 图片：工具栏插入图片、粘贴剪贴板图片、拖入图片文件；图片始终落盘以保持源码整洁（不内联 base64）：已绑定文件时存到文档同目录 `assets/` 用相对路径，未绑定文件时存到插件数据目录 `<userData>/markdown-editor/assets/` 用 `file://` 绝对路径
- 统一以原始 Markdown 插入：AI 文本、生成图片、粘贴 / 拖入等都写入原始 Markdown，由实时预览即时渲染（不再需要区分编辑器模式）
- base64 自动整理：
  - 粘贴含内联 `![](data:image/...;base64,...)` 的 Markdown 文本时，自动把 base64 图片提取落盘并替换为短路径引用，源码不再被超长 base64 撑满
  - 工具栏「整理图片」按钮：一键把当前文档里所有内联 base64 图片转成文件引用（落盘策略同上），转换进编辑器历史可 `Ctrl/Cmd+Z` 撤销；解析或落盘失败的引用保持原样，不丢内容
- AI 助手（走宿主 `mulby.ai`）：两种入口
  - **悬浮工具栏（划词即用，推荐）**：在编辑器里选中文字后，会在选区上方自动弹出一个悬浮工具栏，直接点「润色 / 翻译 / 问一问 / 续写 / 总结 / 自定义 / 生图」即可，无需进面板
    - **`Ctrl/Cmd+J` 快捷唤起**：即使没有选中文字也能随时呼出浮窗——有选区时锚定在选区上、对选区生效；无选区时锚定在光标处、对全文生效（菜单与标题会显示「全文」标记）
    - 「问一问」仅在有选区时出现，作用是让 AI 解释选中的文字是什么、是什么意思（输出解释而非改写，结果以渲染后的 Markdown 阅读视图展示，可复制 / 插入）
    - 翻译会就地展开目标语言下拉、自定义会就地展开提示词输入框（`Ctrl/Cmd+Enter` 快速生成），全部在当前页面完成
    - 流式生成、可随时停止；结果就地预览，有选区时可「替换选区 / 在选区后插入 / 复制 / 重新生成」，无选区（全文）时提供「在光标处插入 / 复制 / 重新生成」，应用进编辑器历史可 `Ctrl/Cmd+Z` 撤销
    - **结果可追问 / 迭代**：出结果后可在同一结果上继续追问微调——一排快捷项（更短 / 更详细 / 更正式 / 更口语 / 换个说法）+ 自由输入框（回车发送），多轮打磨在上一轮输出基础上进行（划词浮窗与完整面板均支持）
    - **替换前 diff 预览**：有选区时结果区可切「对照」，以 before→after 高亮（删除/新增）查看改动后再决定替换
    - **思考过程指示**：模型有推理输出时，正文到达前显示「思考中」并实时展示其推理（不只是转圈）
    - 浮窗会随选区/光标自动定位（空间不足时自动翻到下方并贴边），点击别处或按 `Esc` 关闭；右侧「⤢」可一键展开为完整面板
  - **完整面板**：工具栏 ✦ 按钮或 `Ctrl/Cmd+K` 打开
    - 五种操作：润色、续写、翻译（含目标语言下拉）、总结、自定义指令
    - 润色 / 翻译需要先选中文字；续写 / 总结 / 自定义在无选区时自动处理全文；**空文档时「自定义」可仅按你的指令直接生成内容**（如「写一首关于春天的诗」）
    - **润色 / 翻译带上下文**：会附带选区前后文作为只读参考（不被改写/翻译），让指代、术语、语气更连贯
    - 可在面板里选择 AI 模型（默认用宿主默认模型），选择即持久化记忆；悬浮工具栏复用同一模型设置
  - **AI 生图**：悬浮工具栏「生图」或工具栏生图按钮打开生图对话框（走宿主 `mulby.ai.images`）
    - 两个入口：划词后点「生图」会把选中文字带入提示词；无选区时打开空白提示词由你输入；提示词均可编辑
    - 流式优先（可看到渐进预览、随时停止）；可选生图模型（自动过滤 `image-generation` 模型并记忆所选）
    - 输出尺寸以**比例**呈现，便于理解：`9:16 / 2:3 / 3:4 / 1:1 / 4:3 / 3:2 / 16:9`
    - **每篇文档的生图历史会保留**：生成后即落盘到 `<userData>/markdown-editor/gen-history/` 并按文档记录，关闭再打开对话框仍能看到本文档生成过的图片（即使从未插入）；点历史缩略图可预览、再点「插入文档」或双击缩略图即可插入
    - 点「插入文档」即把图片**落盘**（绑定文件存文档同目录 `assets/` 相对路径，未绑定时存 `<userData>/markdown-editor/assets/` 用 `file://`）并在原光标处插入引用，不覆盖用作提示词的选中文字
    - 未检测到生图模型时给出提示，引导到 Mulby「AI 设置」添加 `image-generation` 模型
  - 未配置 Mulby AI 时给出友好提示，不会崩溃
- 自动草稿保存到插件存储（未绑定文件时），也支持手动保存
- 支持从剪贴板粘贴、从本地打开 Markdown/TXT 文件
- 导出：`.md` / `.html`（内联完整样式）/ `.pdf`（无头渲染）/ `.docx`（Word）
  - Word 导出为**真实表格**（带表头底纹）与**内嵌图片**：相对 `assets/`、`file://` 与内联 base64 图片都会被解码并嵌入文档（统一转 PNG，按页宽等比缩放）；无法解码的图片降级为 `[图片: …]` 文本占位，导出永不因单张图失败
  - HTML / PDF 导出支持**代码高亮**（highlight.js）、**数学公式**（KaTeX，`$…$` / `$$…$$` / `\(…\)` / `\[…\]`）与 **mermaid 图**（```mermaid 代码块渲染为图）：通过按需注入 CDN 资源实现，不增加插件体积；仅在文档含对应内容时注入，需要联网渲染（如需完全离线可改为内置打包）
- 支持通过划词把外部文本直接带入编辑器
