# 桌面宠物生态增强方案

## 目标

桌面宠物已经具备 AI 对话、生活档案、日记、成就、番茄钟、小游戏、前台窗口感知和定时提醒。下一阶段不再单纯增加表情或菜单项，而是把这些能力串成可持续的“宠物生态”：宠物能根据当天发生的事、用户当前工作模式和自身需求状态，做出更自然的表现和提醒。

## 核心设计

- **需求状态**：新增精力、关注、好奇、专注、补水五个 0-100 状态。它们会随时间衰减，也会因聊天、番茄钟、小游戏、提醒和工作模式变化而改变。默认表情和主动搭话会参考这些状态。
- **今日时间线**：记录启动、互动、番茄钟、小游戏、提醒、工作模式变化、日记生成等轻量事件。时间线用于设置页展示和日记生成，不保存前台窗口标题，避免泄露隐私。
- **工作模式**：基于当前应用名和可选窗口标题识别编码、写作、会议、浏览、专注、休闲模式。默认只持久化应用名和模式；窗口标题默认仅作为短期上下文，不写入长期存储。
- **生活节律**：把喝水、休息眼睛、早晚问候和用户自定义提醒统一接入生态事件流。默认保留 45 分钟喝水、90 分钟休息。
- **每日任务与小游戏成长**：每天生成 3 个固定轻量任务：和宠物互动、完成一次番茄钟、玩一次小游戏。小游戏新增积分、答对数和连续答对记录，并同步推进每日任务。

## 数据与接口

- 新增 storage key：
  - `pet-needs-v1`：需求状态。
  - `pet-timeline-v1`：最近事件，限制最大数量。
  - `pet-quests-v1`：每日任务进度。
  - `pet-game-stats-v1`：小游戏积分与 streak。
  - `pet-ecosystem-settings-v1`：生态设置。
- 新增 engine 模块集中处理生态数据，避免 `PetView` 继续承载复杂规则。
- `AIChatController` 增加生态上下文和最近历史读取接口，用于提示词与日记生成。
- 设置页新增“生态”标签，展示需求状态、当前模式、每日任务、小游戏积分和今日时间线。

## 隐私约束

- 默认不把窗口标题写入时间线或任何长期存储。
- 设置页提供“允许窗口标题参与短期上下文”的开关；即使开启，也只传给当前 AI 对话上下文，不进入时间线。
- 剪贴板仍沿用现有敏感内容检查。
- 不改变幽灵 SVG 主体形状、比例、viewBox 或 body path。生态表现通过表情、姿势、动画、移动和气泡完成。

## 验收标准

- 启动后设置页能看到生态状态。
- 聊天、点击、番茄钟、小游戏和提醒会更新需求状态、每日任务和时间线。
- 晚上生成日记时能引用今天的具体事件和最近对话。
- 切换常见应用时能识别工作模式，默认不持久化窗口标题。
- 构建、类型检查和 engine 测试通过。
