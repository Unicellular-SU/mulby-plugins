# 桌面宠物 — 上下文感知与交互增强方案

## 概述

在现有桌面宠物基础上增强 AI 上下文感知能力，让宠物更了解用户状态、行为模式，并基于性格做出个性化回应。

---

## 模块一：亲密度惩罚机制

### 现有奖励
| 行为 | 奖励 |
|------|------|
| 每日签到 | +3 ~ +10（连续签到加成） |
| 互动（5分钟冷却） | +1 |
| 对话（10分钟冷却） | +1 |
| 完成番茄钟 | +5 |

### 新增惩罚
| 场景 | 惩罚 | 触发条件 |
|------|------|---------|
| 长期不签到 | -(gap-2)*2 | 超过2天不打开（已有） |
| 72小时无主动对话 | -5 | 上次 chat() 超过72小时 |
| 连续忽略宠物 | -3 | 宠物主动说话5次用户都未互动 |

### 每日增长上限
- 每日亲密度最多增长 +15，防止刷互动

### 实现位置
- `pet-stats.ts` → `PetStatsController`

## 模块二：AI 上下文注入宠物属性

### 注入内容
在每次 AI 调用时，将宠物状态作为上下文传递：

```
【你和用户的关系】
- 亲密度: {intimacy}/100 ({level})
- 相伴天数: {days}天
- 连续签到: {streak}天
- 今日番茄: {pomodoroToday}个
- 累计互动: {totalInteractions}次
- 心情: {mood}
```

### AI 行为影响
- 亲密度 < 20（冷淡）：宠物更傲娇、不愿多说
- 亲密度 20-50（普通）：正常性格表现
- 亲密度 50-80（温暖）：更热情、话更多
- 亲密度 > 80（亲密）：撒娇/毒舌更猛/分享秘密

### 实现位置
- `ai-chat.ts` → `buildSystemPrompt()` 中追加属性段
- `PetView.tsx` → 初始化时将 statsRef 传递给 AIChatController

## 模块三：剪贴板内容吐槽

### 与翻译的区分
| 条件 | 处理 |
|------|------|
| 英文占比 > 70% | 翻译为中文（已有） |
| 中文占比 > 50% 且 > 10字 | 基于性格吐槽/评论 |
| 其他 | 忽略 |

### 吐槽逻辑
- 冷却时间：5分钟
- 通过 AI 让宠物对剪贴板内容做出性格化评论
- 系统提示词追加：`[用户刚刚复制了一段文字："{内容前50字}..."]`

### 触发条件
- 中文 > 50%
- 内容长度 > 10字
- 与上次内容不同
- 距上次吐槽 > 5分钟
- 非番茄钟期间

### 实现位置
- `PetView.tsx` → 现有 clipboard 定时器中扩展

## 模块四：鼠标行为模式识别

### 行为检测
| 行为 | 检测方式 | 阈值 | 宠物反应 |
|------|---------|------|---------|
| 疯狂移动 | 2秒内位移 > 3000px | 冷却3分钟 | 好奇/吐槽 |
| 频繁点击 | 2秒内 > 5次 mouseDown | 冷却3分钟 | 惊讶/吐槽 |
| 鼠标长时间静止 | 5分钟无 mouseMove | - | 委屈/困惑 |
| 匀速阅读 | 缓慢水平移动 | - | 安静不打扰 |

### 实现方式
- 在 `inputMonitor.onEvent` 中增加行为分析
- 新增 `MouseBehaviorDetector` 类，滑动窗口统计

### 实现位置
- 新文件 `engine/mouse-behavior.ts`
- `PetView.tsx` 中在 inputMonitor 回调中集成

### triggers 扩展
```typescript
triggers: {
  idle: boolean         // 闲置时主动说话
  typing: boolean       // 打字时评论

  lateNight: boolean    // 熬夜提醒
  clipboard: boolean    // 剪贴板内容响应（新）
  mousePattern: boolean // 鼠标行为响应（新）
}
```

### UI 调整
- 在设置页面"触发行为"区域增加两个新开关
- 默认开启
