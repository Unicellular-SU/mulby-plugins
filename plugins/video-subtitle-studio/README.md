# 视频字幕工作台

视频字幕工作台是一个 Mulby 插件，用于从本地视频提取音频，调用用户配置的 ASR 服务生成带时间轴字幕，并支持字幕编辑、AI 翻译和导出。

## 功能

- 导入 `mp4`、`mov`、`mkv`、`webm`、`avi`、`m4v` 视频文件。
- 使用 Mulby 内置 FFmpeg API 抽取 16 kHz 单声道音频。
- 支持 OpenAI / Whisper 类转写接口。
- 支持火山引擎豆包语音极速识别接口。
- 长视频按 8 分钟分片处理，每片保留 5 秒重叠窗口，再合并回全局时间轴。
- 支持字幕文本、起止时间和译文编辑。
- 手动拆分字幕：按文本光标位置、按当前播放头时间、或按词级时间戳（若有）拆成两条；相邻字幕可向上合并。
- 内嵌视频预览：播放头实时高亮对应字幕、字幕列表自动滚动跟随，点击字幕 ▶ 跳转到对应时间，视频上叠加当前字幕。
- 识别过程中逐分片实时刷新字幕列表，左侧字幕条数、原文字数实时更新。
- 分片失败可单独重试：已识别分片保留，失败分片标红并可一键重试，无需重新提取音频。
- 撤销 / 重做：编辑、删除、翻译等操作可回退（`Ctrl/Cmd+Z` 撤销、`Ctrl/Cmd+Shift+Z` 重做）。
- 自动保存草稿：字幕变化后自动写入本机存储，下次打开可一键恢复。
- 字幕时间轴采用虚拟滚动 + 动态行高，数据量大时滚动依然流畅。
- 使用 Mulby AI API 翻译字幕，翻译时关闭 tools、MCP 和 skills。
- 支持导出 `SRT`、`WebVTT`、`JSON`、双语 `SRT`。
- 支持保存 / 导入 `.vssproj` 工程文件，下次可继续编辑。

## 触发方式

在 Mulby 搜索栏输入：

- `视频字幕`
- `字幕`
- `subtitle`

也可以把视频文件作为文件触发入口打开插件。

## ASR Provider 配置

### OpenAI / Whisper

需要填写：

- Base URL，默认 `https://api.openai.com/v1`
- 模型，默认 `whisper-1`
- 语言，如 `zh`、`en`，可留空交给模型判断
- API Key

插件会使用 `response_format=verbose_json` 和 `timestamp_granularities[]=segment` 获取分句时间戳。

### 火山引擎豆包语音

支持两种接口版本，音频都以 base64 随请求上传，无需音频公网 URL，适合本地插件：

| 接口版本 | 适用 | 默认 Resource ID | 调用形式 |
|---|---|---|---|
| 标准版（默认） | 豆包录音文件识别 1.0 / 2.0 | `volc.seedasr.auc`（2.0）/ `volc.bigasr.auc`（1.0） | submit + query 两段式异步轮询 |
| 极速版 | 大模型录音文件极速版 | `volc.bigasr.auc_turbo` | recognize/flash 同步一次返回 |

需要填写（设置弹窗 → 火山引擎）：

- 接口版本：标准版 / 极速版
- 标准版填 Submit URL + Query URL；极速版填 Endpoint（均有默认值）
- Resource ID：必须与你在控制台开通的服务一致
  - 豆包录音文件识别模型 2.0 标准版 → `volc.seedasr.auc`
  - 豆包录音文件识别模型 1.0 标准版 → `volc.bigasr.auc`
  - 极速版 → `volc.bigasr.auc_turbo`
- App ID（请求头 `X-Api-App-Key`）
- Access Token（请求头 `X-Api-Access-Key`）

请求会自动附带 `X-Api-Request-Id`（随机 UUID）与 `X-Api-Sequence: -1`，错误信息会带上 `X-Tt-Logid` 方便排查。

常见报错：

- `45000000 missing X-Api-Request-Id`：缺请求头（插件已自动补齐）。
- `45000030 requested resource not granted`：Resource ID 与控制台开通的服务不一致，或所填 App ID/Access Token 不属于开通该服务的应用。请按上表把 Resource ID 改成你实际开通的版本。

> 当前版本通过客户端 8 分钟分片兼容 1-2 小时长视频；标准版每片走 submit→query 轮询，极速版每片同步返回。

## 长视频策略

长视频不能直接一次性提交给所有 ASR 服务。插件采用以下策略：

1. 先读取视频时长。
2. 按 8 分钟生成音频分片，每片向前保留 5 秒重叠窗口。
3. 每片单独调用 ASR，拿到片内时间戳。
4. 将每条字幕的时间加上分片全局偏移。
5. 删除重叠窗口中的重复字幕。
6. 合并为统一字幕时间轴。

这种策略适合 1-2 小时电影、课程、会议录制等长音视频。它比直接上传整段视频更稳定，也便于失败后定位重试。

## 工程文件

右上角「导入工程 / 保存工程 / 另存为」按钮（设置按钮旁）支持保存与导入工程，便于跨会话继续编辑：

- 「保存工程」会把当前视频路径、视频时长、全部字幕（含译文、时间轴、词级时间戳）以及 Provider / 目标语言等偏好写入一个 `.vssproj` 文件（本质是结构化 JSON）。
  - 首次保存会让你选择保存位置；之后再点「保存工程」会**直接覆盖**当前已绑定的工程文件，不再弹窗。
  - 绑定工程文件后，编辑过程中的自动保存也会写回该文件（同时仍保留本机草稿作为兜底）。
- 「另存为」总是弹窗让你选择新位置，并把后续保存绑定到新文件。
- 「导入工程」会读取 `.vssproj`（也兼容 `.json`），恢复字幕列表与时间轴并绑定该文件，无需重新识别即可继续编辑、翻译、导出。

工程文件格式：

```json
{
  "format": "video-subtitle-studio/project",
  "version": 1,
  "savedAt": "2026-06-05T12:00:00.000Z",
  "videoPath": "/path/to/video.mp4",
  "durationMs": 7200000,
  "cues": [{ "id": "cue-1", "startMs": 0, "endMs": 2000, "text": "...", "translation": "..." }],
  "meta": { "provider": "openai", "targetLanguage": "英文" }
}
```

工程文件不包含任何 API Key / Access Token，密钥始终只保存在 Mulby 本机加密存储中。

## 拆分与合并字幕

识别出来的字幕过长时，点击该条字幕右上角的 ✂️ 展开拆分条，支持三种拆分方式：

| 方式 | 操作 | 时间切分 | 文本切分 |
|---|---|---|---|
| 按文本光标 | 把光标点到文本框里想断开的位置，点「在文本光标处」 | 按字符比例换算 | 在光标处（英文按词边界对齐） |
| 按播放头 | 播放视频到目标时刻，点「在播放头」（播放头需落在本条时间范围内） | 用当前播放时间 | 按时间比例换算 |
| 按词级时间戳 | 若该字幕带词级时间戳，点某个词，从该词前断开 | 用该词的真实开始时间 | 用真实词边界，最精确 |

相邻字幕可通过 ▲ 按钮与上一条合并。拆分、合并都会进入撤销栈，可用 `Ctrl/Cmd+Z` 回退。

> 译文在拆分时只保留在前半条，避免重复；合并时两条译文会拼接。

## 隐私说明

视频文件保留在本机。插件会在本机临时目录生成音频分片，并把音频分片发送到用户选择的 ASR Provider。API Key 使用 Mulby 加密存储保存在本机。处理完成后，插件会尝试清理临时音频文件。

## Mulby 手动验收

1. 插件能通过 `视频字幕` 打开独立窗口。
2. 导入一个短视频后能读取时长。
3. 配置 OpenAI 或火山引擎凭证后能生成字幕。
4. 字幕可以编辑时间和文本。
5. Mulby AI 翻译能生成译文。
6. SRT、VTT、JSON 和双语 SRT 能保存到本地。
7. 处理结束后临时文件不会留在工作目录中。
8. 「保存工程」能生成 `.vssproj` 文件，「导入工程」能恢复字幕继续编辑。
9. 内嵌视频播放时，当前字幕在列表中高亮并自动滚动；点击字幕 ▶ 能跳转到对应时间。
10. 某个分片识别失败时，其余分片字幕仍保留，失败分片可单独重试成功。
11. 编辑/删除字幕后 `Ctrl/Cmd+Z` 能撤销、`Ctrl/Cmd+Shift+Z` 能重做。
12. 关闭并重新打开插件后，能看到草稿恢复提示并恢复上次未保存的字幕。
13. 点击字幕的 ✂️ 能在文本光标处 / 播放头 / 词级时间戳处把字幕拆成两条；▲ 能把字幕与上一条合并。
